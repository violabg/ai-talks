# Style A — Illustrative

**Think illustration, not diagram.** Draw a small picture — a character, an object, a scene — not a technical schematic. Best covers look like tiny hand-drawn text illustrations.

## Reference example (rubber duck cover)

```
      __
    <(o )____A
     (  ._>  /       __
  ~~~~`-----'~~~~  <(o )____A
          ~~~~~~    (  ._>  /
                 ~~~~`-----'~~~~
      dual-model review
```

A **drawing of rubber ducks** — tells a story, has personality, uses simple ASCII (`/`, `\`, `_`, `~`, `<`, `>`, parentheses) to form recognizable shapes.

Character selection is model-driven: pick symbols that best express topic and chosen style. Not restricted to a fixed set — Unicode/box-drawing glyphs allowed when they improve clarity.

## Design principles

1. **Draw objects and characters, not diagrams** — a robot face, an animal, a tool, a scene with depth. "Tiny illustration" not "flowchart".
2. **Use the most suitable characters** — basic ASCII is often enough; box-drawing / Unicode allowed for legibility, style, or topic fit.
3. **Tell a visual story** — the art should have a subject that relates to the topic. A duck reviewing code beats a box labeled "review".
4. **Add environmental details** — water ripples (`~~~~`), ground lines, shadows, motion lines, small decorative elements that give the scene life.
5. **Include a short caption** — 1–3 word label in muted color at the bottom to anchor the meaning.
6. **Fits the card gracefully** — choose width/height dynamically based on concept and detail; avoid hard caps; optimize for balance without overwhelming metadata.
7. **Visual weight balance** — not too dense, not too sparse. Leave breathing room around the subject.

## Topic inspiration (illustrative)

- **Testing / feedback loops**: bug under a magnifying glass, lab flask bubbling, cat batting at a failing test
- **CLI tools**: rubber duck at a terminal, wizard casting commands, toolbox with tools spilling out
- **Architecture / patterns**: tower of building blocks, bridge between islands, tree with branching roots
- **Refactoring / migration**: caterpillar becoming a butterfly, house being renovated, before/after sketches
- **Orchestration / multi-model**: conductor with a baton, birds flying in formation, puppet master with strings
- **Configuration / setup**: control panel with levers, key fitting a lock, compass pointing north
- **AI agents**: robot with a thought bubble, brain with lightning, octopus juggling tools
- **General**: animals, space scenes, weather, landscapes, everyday objects — anything that creates a visual metaphor
