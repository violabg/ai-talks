# Style B — Technical

**Think structured schematic.** Clean, precise diagram using box-drawing characters — terminal windows, flowcharts, module layouts. Informational and developer-oriented.

## Design principles

1. **Visually recognizable** — glance = sense of article topic.
2. **Use the character language that best communicates structure** — box-drawing and symbolic glyphs encouraged, not mandatory; pick what matches the topic.
3. **Fits the card gracefully** — dimensions dynamic to topic complexity and readability; no fixed width/height limits.
4. **Visual weight balance** — not too dense, not too sparse.

## Topic inspiration (technical)

- **Testing / feedback loops**: terminal output with checkmarks/crosses, test runner UI, circular arrows
- **CLI tools**: terminal window frame with prompt + output, command pipeline
- **Architecture / patterns**: module boxes connected by arrows, layered stack diagram
- **Refactoring / migration**: before→after diff view, transformation arrows
- **Orchestration / multi-model**: network of connected nodes, message passing diagram
- **Configuration / setup**: config file snippet, toggle switches, slider bars
- **AI agents**: robot/brain icon, thought bubbles, tool invocation chains
